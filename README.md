# knowledge-db

## 使い方（MVP）
- どの端末からでも GitHub Issue を作成（テンプレ: Idea）
- label `inbox` が付くと Actions が `knowledge/00_inbox/` に md を生成して自動コミット
- Cursor で整形してカテゴリに移動、type を concept/playbook/decision/project に昇格して push

## ナレッジの型
- concept: 概念
- playbook: 手順化された型
- decision: 意思決定ログ
- project: 施策ログ

## 整形ルール（推奨）
- 00_inbox の note を、必要に応じて concept に昇格
- validated は「一度使って効いた」ものだけ
